CONTENT PROTECTION — REALISTIC LIMITS

This package includes:
- context-menu blocking on protected previews
- drag blocking
- Ctrl/Cmd+S and Ctrl/Cmd+U deterrence
- watermarks on portfolio previews
- frame-busting deterrence
- a restrictive Content Security Policy
- no third-party scripts or trackers

These measures deter casual copying only. Any public browser content can still be captured through screenshots, developer tools, network tools, browser extensions, or automated requests.

For stronger protection:
1. Publish only compressed, watermarked previews.
2. Never upload editable PPTX, PSD, AI, AE, Premiere, preset, code, or saleable source files to the public repository.
3. Keep originals offline and in private cloud storage.
4. Deliver paid/restricted files through an authenticated service with expiring signed links.
5. Add visible copyright and licence terms.
6. Use YouTube for video previews and disable downloads where available, while recognizing screen recording is still possible.
